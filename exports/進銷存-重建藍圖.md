# 進銷存重建藍圖

目標：保留原始 Access 資料，重建成一套結構清楚、可維護、可報表化的新版進銷存。

## 重建順序

1. 主檔
   - 商品
   - 客戶
   - 供應商
   - 倉庫
   - 商品類別
2. 進貨
   - 進貨單主檔
   - 進貨單明細
   - 入庫與批號追蹤
3. 銷貨
   - 銷貨單主檔
   - 銷貨單明細
   - 發票/收款欄位
4. 庫存
   - 庫存總表
   - 庫存異動流水
   - 預扣/待扣
   - 倉庫調撥
5. 報表
   - 出貨單
   - 月銷統計
   - 商品排名
   - 客戶排名
   - 庫存低於安全量
6. 操作介面
   - 首頁儀表板
   - 主檔維護
   - 進貨輸入
   - 銷貨輸入
   - 查詢與列印

## 舊資料對照

- `商品基本資料` -> `Products`
- `客戶基本資料` -> `Customers`
- `供應商基本資料` -> `Suppliers`
- `倉庫基本資料表` -> `Warehouses`
- `商品類別表` -> `ProductCategories`
- `進貨基本資料` -> `Purchases` / `PurchaseLines`
- `銷貨基本主檔` -> `Sales` / `SaleLines`
- `庫存待扣一覽表` -> `InventoryPending`
- `會計庫存基本主表` / `會計庫存基本副表` -> `InventoryLedger`

## 先做的新版核心

### 主檔欄位原則

- 每張表都要有單一主鍵
- 名稱欄位與代碼欄位分開
- 金額用數值欄位
- 數量與庫存用數值欄位
- 備註與說明保留長文字

### 交易表原則

- 主檔只放單據層級資料
- 明細表只放品項層級資料
- 所有庫存變化都寫入異動流水
- 出貨與收款都保留可追溯欄位

## 驗收條件

- 可新增/修改商品、客戶、供應商、倉庫
- 可建立進貨單與銷貨單
- 庫存會即時更新
- 可輸出出貨單
- 可查每月銷量與商品排名
